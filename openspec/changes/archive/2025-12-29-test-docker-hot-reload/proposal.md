# 提案：测试 Docker 卷挂载代码热重载功能

## 问题背景

在开发过程中发现 Docker 卷挂载后代码修改似乎没有触发热重载，容器日志中也没有观察到重启记录。需要验证并诊断 Spring Boot DevTools 在 Docker 环境中的热重载行为。

## 变更目标

1. **诊断当前问题**：确认 Spring Boot DevTools 是否正确配置和工作
2. **创建测试端点**：添加一个简单的测试 Controller，用于验证代码变更是否生效
3. **验证热重载**：通过修改测试端点代码，观察容器是否检测到变更并重新加载
4. **清理测试代码**：验证完成后，在用户许可下删除测试文件

## 影响范围

- **新增文件**：
  - `src/main/java/org/ruikun/controller/HotReloadTestController.java`（临时测试文件）
- **无修改**：不修改任何现有功能代码
- **无影响**：不影响现有的业务逻辑

## 实现方案

### 技术方案

1. 添加 spring-boot-devtools 依赖（如果缺失）
2. 创建一个简单的测试端点 `/api/test/hot-reload`
3. 通过修改端点返回值来验证热重载
4. 观察 Docker 日志确认 DevTools 是否工作

### 测试步骤

1. 创建测试 Controller
2. 访问测试端点获取初始响应
3. 修改测试端点代码
4. 再次访问端点，观察响应是否变化
5. 检查 Docker 日志是否有重启记录
6. 根据结果诊断问题

## 风险评估

- **风险等级**：极低
- **影响范围**：仅添加一个临时测试端点，不影响现有功能
- **回滚方案**：直接删除测试文件即可

## 后续行动

完成测试后，根据诊断结果：
- 如果 DevTools 未工作：提供修复建议（如调整 Dockerfile 或 pom.xml 配置）
- 如果 DevTools 正常工作：文档化热重载的使用方法
- 清理：在用户确认后删除测试文件
