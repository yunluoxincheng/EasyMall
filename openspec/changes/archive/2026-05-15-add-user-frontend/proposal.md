## Why

EasyMall 后端已完成订单状态机、库存锁定、支付幂等、MQ 延迟关单、优惠券生命周期和积分流水等核心交易链路，管理端前端也已实现。但项目缺少用户端前端，无法演示完整的"浏览商品→购物车→下单→支付→订单"购物流程。用户端前端是让 EasyMall 达到"可完整演示"状态的关键一步，也是阶段 8 文档/测试/部署前必须补齐的能力。

## What Changes

- 在现有 `easymall-frontend` 项目中添加用户端路由与布局，与管理端共存于同一 Vue3 项目
- 添加用户端公共布局（顶部导航、底部 Footer、用户菜单）
- 添加商品浏览页面（商品列表、分类筛选、商品详情、热门/新品推荐）
- 添加购物车页面（购物车列表、数量修改、全选/批量选择、结算跳转）
- 添加下单与支付流程页面（订单确认、优惠券选择、提交订单、模拟支付）
- 添加用户中心页面（个人信息编辑、密码修改、我的订单、订单详情、收货确认）
- 添加收藏功能页面（收藏列表、收藏/取消收藏）
- 添加评论功能页面（商品评论列表、发表评论、我的评论管理）
- 添加优惠券页面（优惠券中心/领取、我的优惠券）
- 添加积分与会员页面（积分记录、积分商品兑换、签到、会员等级、会员折扣）
- 添加用户注册与登录页面
- 添加用户端 API 模块与类型定义
- 添加用户端路由守卫（未登录跳转登录页）

## Capabilities

### New Capabilities

- `user-layout`: 用户端公共布局（顶部导航栏、底部 Footer、用户菜单下拉、购物车图标入口）
- `user-auth`: 用户注册、登录、登出流程
- `product-browsing`: 商品列表浏览、分类筛选、关键词搜索、商品详情、热门/新品推荐、相关商品
- `cart-management`: 购物车列表、添加商品、修改数量、删除、全选/批量选择、结算跳转
- `order-flow`: 订单确认页（地址/优惠券选择）、提交订单、模拟支付、订单列表、订单详情、取消订单、确认收货
- `user-profile`: 个人信息查看与编辑、密码修改
- `favorite-management`: 收藏商品、取消收藏、收藏列表
- `comment-interaction`: 查看商品评论、发表评论、我的评论管理
- `coupon-center`: 优惠券模板列表/领取、我的优惠券列表、下单时可用优惠券选择
- `points-and-member`: 积分记录、积分商品浏览与兑换、每日签到、会员等级与折扣查看

### Modified Capabilities

（无已有 spec 的行为变更，用户端全部为新增能力）

## Impact

- **前端代码**: `easymall-frontend` 项目新增约 15+ 个页面组件、对应的 API 模块和类型定义
- **路由**: Vue Router 新增用户端路由组（以 `/` 前缀），与管理端路由（以 `/admin` 前缀）共存
- **API 调用**: 新增用户端 API 模块，调用后端已有的 `/api/user`、`/api/product`、`/api/cart`、`/api/order`、`/api/payment`、`/api/coupon`、`/api/favorite`、`/api/comment`、`/api/points`、`/api/member`、`/api/signin`、`/api/category` 等接口
- **依赖**: 不新增 npm 依赖，复用现有 Naive UI、Pinia、Vue Router、Axios
- **后端**: 无后端代码变更，所有用户端 API 已就绪
