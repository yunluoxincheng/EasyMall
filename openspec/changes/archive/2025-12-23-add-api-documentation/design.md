# API 文档设计文档

## Context

当前项目有 92 个 API 接口，分布在以下模块：
- 用户模块（注册/登录/用户信息）
- 商品模块（查询）
- 分类模块（查询）
- 购物车模块
- 订单模块
- 评论模块
- 收藏模块
- 会员模块
- 签到模块
- 积分模块
- 积分兑换模块
- 后台管理模块（7个子模块）

## 文档结构

### 1. 概述部分
- API 基础信息
- 通用响应格式
- 认证方式
- 错误码说明

### 2. 接口分类
按模块组织，每个模块包含：
- 模块说明
- 接口列表
- 每个接口的详细信息

### 3. 接口详情格式
```markdown
### 接口名称

**请求**: `METHOD /api/path`

**说明**: 接口功能描述

**权限**: 公开/需认证/管理员

**请求参数**:
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| ... | ... | ... | ... |

**响应示例**:
```json
{
  "code": 200,
  "message": "操作成功",
  "data": { ... }
}
```
```

## 数据模型

### 统一响应格式
```json
{
  "code": 200,
  "message": "操作成功",
  "data": {}
}
```

### 分页响应格式
```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "total": 100,
    "records": [],
    "pageNum": 1,
    "pageSize": 10,
    "pages": 10
  }
}
```

## 认证方式

使用 JWT Token 认证：
1. 调用登录接口获取 token
2. 后续请求在 Header 中携带：`Authorization: Bearer {token}`
