# 提案：添加 API 接口文档

## 问题陈述

当前项目缺少正式的 API 文档，只有：
1. `test-api.http` 文件 - 主要用于开发测试
2. OpenSpec 规范文件 - 面向 AI 助手的规范定义

对于前端开发者、API 使用者和项目维护者来说，缺少一份清晰、易读的 API 文档。

## 目标

创建一份完整的 API 接口文档，包括：
1. 所有接口的路径、方法、参数、响应
2. 认证和权限说明
3. 数据模型定义
4. 错误码说明
5. 调用示例

## 文档格式选择

考虑到项目特点，创建以下格式的文档：
- **Markdown 格式** (`docs/API.md`) - 易读易维护，支持 Git 版本控制
- 内容基于现有的 `test-api.http` 和代码实现

## 影响范围

- 新增 `docs/API.md` 文档
- 不修改任何现有代码
- 文档包含 92 个接口的完整说明

## 验收标准

1. 文档涵盖所有 92 个接口
2. 每个接口包含：路径、方法、参数、响应示例
3. 包含认证和权限说明
4. 包含通用数据模型和错误码说明
