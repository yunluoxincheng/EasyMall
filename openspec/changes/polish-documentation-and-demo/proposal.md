## Why

EasyMall 已完成路线图阶段 1-8A 的全部重构工作（工程结构、订单状态机、库存锁定、支付幂等、MQ 事件驱动、优惠券生命周期、积分幂等、管理端/用户端前端、测试与 CI），但当前文档仍停留在早期课程作业阶段的描述，未能反映项目实际的企业级交易链路设计。项目需要一个完整的文档体系来支撑简历展示、毕设答辩和面试讲解，让别人能理解"为什么要这样设计"而不仅仅是"做了什么"。

## What Changes

- 重写根目录 README，突出企业级交易链路设计亮点（订单状态机、库存锁定、支付幂等、MQ 延迟关单等）
- 新增架构概览文档（系统架构图、后端模块化架构、前端架构、部署架构）
- 新增核心业务设计文档（订单状态机、库存模型、支付系统、MQ 事件驱动、优惠券生命周期、积分流水）
- 新增数据库设计文档（ER 图、核心表说明、索引策略）
- 新增前端开发指南（管理端、用户端）
- 重写部署指南，对齐最新的前后端独立 Docker 部署方案
- 新增演示指南（默认账号、管理端/用户端演示路径、核心交易链路演示步骤）
- 整理现有 `docs/` 目录，淘汰过时内容，统一文档结构

## Capabilities

### New Capabilities
- `readme-rewrite`: 根目录 README 重写，突出项目亮点和技术栈全貌
- `architecture-docs`: 系统架构文档（后端架构、前端架构、部署架构）
- `business-design-docs`: 核心业务设计文档（订单状态机、库存模型、支付系统、MQ 事件、优惠券生命周期、积分流水）
- `database-design-doc`: 数据库设计文档（ER 图、核心表、索引策略）
- `frontend-guide`: 前端开发与构建指南
- `deployment-guide-rewrite`: 部署指南重写，对齐前后端独立 Docker 部署方案
- `demo-guide`: 演示指南（默认账号、管理端/用户端演示路径、核心交易链路演示步骤）

### Modified Capabilities
- `readme-documentation`: 合并到 readme-rewrite，更新展示需求
- `cloud-deployment-guide`: 部署指南需要完全重写以反映新的前后端独立 Docker 结构

## Impact

- 仅影响 `docs/` 目录和根目录 `README.md`
- 不涉及任何业务代码变更
- 不涉及数据库 schema 变更
- 不影响 API 接口
