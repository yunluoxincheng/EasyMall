# 演示指南

## 默认账号

| 角色 | 用户名 | 密码 | 说明 |
|------|--------|------|------|
| 管理员 | admin | admin123 | 后台管理端使用 |
| 普通用户 | 需注册 | 自定义 | 用户端使用，注册后自动获得新人专享券 |

## 系统启动

```bash
# 1. 启动后端（MySQL + Redis + RabbitMQ + Spring Boot）
cd easymall-backend
docker compose up -d

# 2. 启动前端
cd easymall-frontend
docker compose up -d
```

确认所有服务正常运行：
- 前端：http://localhost:5173（本地开发）或 http://localhost:80（Docker）
- 后端 API：http://localhost:8080/api
- RabbitMQ 管理：http://localhost:15672

## 管理端演示路径

### 1. 管理员登录

- 打开前端首页
- 使用 `admin / admin123` 登录
- 登录后自动进入管理后台

### 2. 商品管理

- **商品列表**：查看所有商品，支持按名称、分类、状态筛选
- **新增商品**：填写商品信息，上传图片，设置库存和价格
- **库存管理**：修改商品库存（同步更新 `inventory` 表）
- **上下架**：控制商品在前台的可见状态

### 3. 订单管理

- **订单列表**：查看所有订单，支持按订单号、用户、状态筛选
- **订单详情**：查看订单的商品明细、收货信息、支付信息
- **状态操作**：备货（`PAID → WAITING_SHIPMENT`）→ 发货（`WAITING_SHIPMENT → SHIPPED`）、取消订单
- **观察要点**：订单状态流转严格遵循状态机规则

### 4. 优惠券管理

- **创建优惠券模板**：设置类型（固定金额/百分比折扣）、使用门槛、有效期
- **上下架**：控制优惠券的领取状态
- **使用记录**：查看所有优惠券的使用和返还日志

### 5. 评论审核

- **评论列表**：查看所有评论，支持按状态筛选
- **审核操作**：通过/拒绝评论
- **商家回复**：对评论进行官方回复

### 6. 其他管理

- **会员等级**：配置等级积分范围和折扣
- **分类管理**：管理商品分类（支持多级）
- **积分商品**：管理积分兑换商品

## 用户端演示路径

### 1. 注册/登录

- 注册一个新用户账号
- 注册成功后自动获得新人专享券（可在"我的优惠券"中查看）

### 2. 浏览商品

- 首页查看热门商品和新品推荐
- 点击分类筛选商品
- 搜索商品（MySQL 全文索引 + Redis 缓存）
- 查看商品详情页（描述、价格、评论）

### 3. 购物车

- 将商品加入购物车
- 修改商品数量（自动校验库存）
- 勾选要结算的商品

### 4. 下单

- 点击结算进入订单确认页
- 填写收货信息
- 选择优惠券（系统自动计算会员折扣 + 优惠券优惠）
- 提交订单（触发：锁定库存 → 锁定优惠券 → 创建支付单 → 发布延迟关单消息）

### 5. 支付

- 进入支付页面，查看支付单信息
- 点击"模拟支付"
- 支付成功后（触发：确认库存 → 使用优惠券 → 订单状态变为已支付）

### 6. 查看订单

- 查看订单列表，观察状态变化
- 查看订单详情

### 7. 会员功能

- **每日签到**：签到获得积分，连续签到有额外奖励
- **积分记录**：查看积分变动历史
- **积分兑换**：用积分兑换商品或优惠券
- **会员等级**：查看当前等级和下一等级的积分要求

### 8. 社交功能

- **商品评论**：对已完成的订单商品进行评价（评分 + 文字 + 图片）
- **商品收藏**：收藏/取消收藏商品

## 核心交易链路亮点演示

### 1. 库存锁定验证

1. 记住某个商品的当前库存数量
2. 下单购买该商品（选择多个数量）
3. 在数据库中检查 `inventory` 表：
   - `available_stock` 减少对应数量
   - `locked_stock` 增加对应数量
   - `product.stock` 同步更新为新的 `available_stock`
4. 取消订单，观察库存恢复

### 2. 超时自动取消

1. 创建一个订单（不支付）
2. 等待超时时间到达（默认 30 分钟，可在配置中调整）
3. 观察 RabbitMQ 管理面板中延迟队列消息的流转
4. 超时后：
   - 订单自动取消（`PENDING_PAYMENT → CANCELLED`）
   - 库存自动释放（`locked_stock → available_stock`）
   - 优惠券自动返还（`LOCKED → UNUSED` / `EXPIRED`）
   - 支付单自动关闭

### 3. 支付幂等

1. 创建订单并进入支付页面
2. 快速多次点击"支付"按钮
3. 观察：
   - 只有第一次点击成功触发支付流程
   - 后续点击返回"支付状态不允许此操作"
   - 支付单状态严格遵循 `WAITING_PAY → PAYING → PAID`，不允许跳过
4. 检查 `payment_callback_log` 表，每次请求都有记录

### 4. 优惠券锁定与返还

1. 在"我的优惠券"中找到一张可用优惠券
2. 下单时选择该优惠券
3. 下单后，优惠券状态变为"已锁定"
4. 取消订单
5. 观察：
   - 如果优惠券未过期：状态从"已锁定"恢复为"未使用"（`UNUSED`）
   - 如果优惠券已过期：状态变为"已过期"（`EXPIRED`）
   - `coupon_usage_log` 中记录了返还操作
   - 未过期的优惠券可以在下次下单时继续使用
